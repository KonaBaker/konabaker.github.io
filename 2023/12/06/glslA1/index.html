2.6 blending colors

codepen链接2.6

  • gl_FragCoord
    GLSL提供了gl_FragCoord这是像素的坐标vec4类型。
    x,y是(0,0)~(width,height)的坐标值。z是深度，通过顶点着色器插值得到的结果。
    主要用于获得片元的窗口坐标和片元深度值
    和屏幕不一样，默认情况下原点(0,0)在左下角。

  • mix方法
    混合颜色可以使用GLSL提供的mix方法。
    mix(x,y,a) x,y代表混合的两种颜色,a代表一种权重(float),多大程度上选择x作为返回值。
    x,y 可以是float,vec2,vec3或者vec4

根据像素的纵坐标发生变化

2.7 uniforms and varyings

codepen链接同上

  • uniforms
    传递信息的媒介,可以是多种类型,甚至是图片
  • varying
    从顶点着色器到片段着色器之间传递的变量
    传递的varying uv坐标是三角形插值后的结果

这个v_uv和gl_FragCoord得到的uv的区别是

  • gl_FragCoord是片元的位置,然后经过分辨率矫正后的坐标,随着调整屏幕大小会变化。右上角是屏幕大小
  • v_uv是直接得到的0~1范围的坐标。右上角是(1,1) 当改为传递v_position后。由于是model的世界坐标,左下角x,y均为负数，传递颜色的时候被钳制为0,所以呈现黑色。

3.7 Tiling

  • fract(float) 返回值是去掉整数部分的小数
    平铺 只看水平方向,假设铺设数量是c,且从0开始编号,那么每一块的长度就是$1/c$,现在有一个uv,假设其横坐标落在编号为k的块内.则有 $k/c < uv < (k+1)/c$。同时乘$c$, 有$k < uv × c < k+1 $ ①
    $(uv-k/c)/(1/c)$ 是在小块中的相对位置,即渲染原本大块中的位置。最终为$uv×c - k$ ,因为①所以等价于$uv×c$的小数部分。
    垂直方向同理。
    所以有了代码中的 vec2 p = fract(v_uv * tilecount);

atan

  • 单参数形式： float atan(float y) 或 vecN atan(vecN y)

计算y的反正切，返回的角度范围是 $(-\pi/2)$ 到 $(+\pi/2)$。

  • 双参数形式： float atan(float y, float x) 或 vecN atan(vecN y, vecN x)

计算$y / x $的反正切，并且考虑x和y的符号来确定象限。返回的角度范围是 $(-\pi)$ 到 $(+\pi)$。
